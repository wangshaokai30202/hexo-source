身为一个技术宅（哈哈，其实是个小白），有事没事总爱写点东西。有没有用先不说，确实能学到一些东西。而且学习的过程也很有趣。  
之前做个人博客一直是基于宝塔面板内WordPress无脑搭建，对于我这种小白特别友好，但是太占用资源，且WordPress的主题过于无聊，最重要的是稍微处理不当，就会出现访问延迟高，网站加载慢的情况。除此之外，服务器费用也是一笔不小的支出，虽然国内一些云服务器厂商每年会推出一些特价轻量云服务器，比如阿里云，腾讯云，华为云...但是国内服务器需要备案，之前有过一次备案经历——堪比玄学，整个过程属实很难受，我用的域名是海外购买的，按理来说应该无需备案才对，结果发现居然服务器绑定域名之后强制让我备案，而且需要先转移域名，才可以备案。  
海外服务器的话要么延迟太高，要么价格又太贵，总找不到一个折中方案。  
于是干脆一不做二不休，用最原始的办法，托管GitHub，不过所有的东西都要需要从头学起，就当打发时间了吧。  


##Markdown语法基础入门  
**Hexo**写博客其实不需要掌握很多复杂语法，Markdown 规则简单又好记。下面是最常用、也最够用的一套写法。  
1. 标题：使用 # 来表示标题，数量越多级别越低：
    ```
    # 一级标题
    ## 二级标题
    ### 三级标题
    ```

2. 加粗、斜体、删除线:
    ```
    **加粗**
    *斜体*
    ~~删除线~~
    ```

3. 段落和换行:
    紧密换行：
    在上一行末尾加两个空格，再回车：  
    ```
    第一行␣␣
    第二行
    ```
    分段换行：
    中间空一行：
    ```
    第一段

    第二段
    ···

4. 列表
    无序列表：
    ```
    - 项目 1
    - 项目 2
    - 项目 3
    ```

    有序列表：
    ```
    1. 第一步
    2. 第二步
    3. 第三步
    ```

5. 插入链接  
[显示文字](https://你的链接)
例：[我的博客主页](https://www.akaiblogs.top)  
6. 插入图片  
用 R2 图床 或 GitHub 图床：![替代文字](图片链接)  
7. 插入代码块（最常用）  
    ```
    **```语言名(不标记也可以，中间随便写)
    代码内容
    ```**
    ```
例：  
    ```python
    print("Hello Hexo!")
    ```

8. 引用  
    ```
    > 这是一段引用文字
    ```
9. 表格  
```
| 姓名 | 年龄 | 城市 |
| --- | --- | --- |
| 小王 | 22 | 齐齐哈尔 |
| 小李 | 23 | 哈尔滨 |
```
